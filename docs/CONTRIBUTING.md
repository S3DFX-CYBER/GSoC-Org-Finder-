# Contributing (Docs Mirror)

See root `CONTRIBUTING.md` for full operational details.

## Strict PR Requirements
- Link valid issue.
- Use PR template.
- Classify one program only.
- Keep changes related and scoped.
- Use DCO sign-off (`git commit -s`).

## Strict Issue Requirements
- Include context, reproduction, expected, and actual behavior.
- Low-information issues receive `needs-more-info` first.
- Spam may be closed automatically.

## Assignment Policy
- Single contributor per issue.
- Approval via mentor/maintainer only.
- Reassignment requires explicit release or inactivity workflow.
