# GSSOC CONTRIBUTOR GUIDE

## Governance Summary
- Use assignment commands only.
- Follow strict PR + issue validators.
- Respect anti-spam and duplicate checks.

## Common Validation Failures
- Missing issue link
- Invalid program classification
- Duplicate/unrelated PR
- Low-quality issue report
- DCO/sign-off missing

## How to Recover
1. Read sticky bot comment.
2. Edit issue/PR with requested fixes.
3. Push updates.
4. Ask for reopen if previously closed.

## Program Path
- GSSOC: mentor-driven, leaderboard-scored, assignment-based.
- NSOC: reviewer-driven with structured flow.
- GENERAL: standard OSS flow without program scoring.
