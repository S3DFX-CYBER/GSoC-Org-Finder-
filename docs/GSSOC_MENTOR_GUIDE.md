Welcome to the **GirlScript Summer of Code 2026** mentor guide!

As a mentor for FindMyGSoC, you play a critical role in maintaining our vanilla codebase philosophy, guiding contributors, conducting code reviews, and managing assignment flows. Your contributions are vital to maintaining a high standard of code quality across the repository.

---

## 📋 Key Mentor Responsibilities

*   **Timely Reviews** — Aim to review assigned PRs or issue requests within **24 hours** to prevent bottlenecks.
*   **Constructive Feedback** — Write clear, encouraging, and actionable review feedback to support contributor growth.
*   **Enforce Zero-Build Standards** — Block any PR that introduces client frameworks or unnecessary npm dependencies.
*   **Audit for Spam & AI Slop** — Politely request clarification on PRs containing autogenerated boilerplate, blind copy-pasting, or duplicate implementations.

---

## 🤖 Automated Mentor Systems & Registration

### Registration
Mentors are registered in our reviewer list at:
```json
.github/reviewers/gssoc-mentors.json
```

### 🕒 The 24-Hour Rotation System
To keep contributions moving and prevent bottle-necks:
*   Mentors are assigned dynamically based on activity.
*   If an assigned mentor does not approve, request changes, or comment on a PR within **24 hours**, the automated bot rotates them out and selects another active mentor.

## 🚦 PR Review Pipeline (3-Stage Workflow)

Mentors should only review PRs that have successfully passed the automated Stage 1 checks.

| Stage | Process | Action / Responsibility |
| :--- | :--- | :--- |
| **Stage 1: Automated Checks** | DCO signature, format verification, duplicate checks, and **TENET AI Agent** security diff scans. | Managed by GitHub workflows. Skip review if `stage-1-approved` label is missing. |
| **Stage 2: Mentor Review** | Human evaluation of features, responsiveness, styling, and vanilla design. | **Mentor Responsibility.** Comment `/approve-pr` or `/lgtm` upon successful review. |
| **Stage 3: Project Admin Gate** | Final checks, administrative merging, and update to public leaderboards. | Handled by **@S3DFX-CYBER** after Stage 2 approval. |

### 🏷️ Stage Labels Reference
*   `needs-mentor-review` — A waiting PR ready for your review.
*   `mentor-review-requested` — Currently assigned to a mentor.
*   `mentor-approved` — Successfully approved at Stage 2.
*   `pa-review-required` — Shifted to the final Project Admin merge gate.

## 🏆 Mentor Approval Commands

### Approving an Assignment Request
When pinged to approve a contributor assignment, verify their eligibility and comment:
```text
/approve-assignment @username
```

### Approving a Pull Request
After validating code correctness, performance, and cross-viewport responsiveness:
```text
/approve-pr
```
*or*
```text
/lgtm
```
*(GitHub's formal Approve review action also works).*

## 🛡️ Escalation Protocols

Escalate concerns directly to the Project Admin (**@S3DFX-CYBER**) if you encounter:
*   Vulnerabilities or severe security risks in a PR diff.
*   Inappropriate or abusive behavior violating our [Code of Conduct](../CODE_OF_CONDUCT.md).
*   Obvious PR farming, duplicate submissions, or spam attempts.
