# Review Guidelines

## Standards
- Verify correctness, scope, and issue alignment.
- Require meaningful test/validation notes.
- Reject unrelated file churn.

## Priority Policy
- P0: security/core infra
- P1: high-impact feature
- P2: standard contribution
- P3: minor low-risk
- P4: docs/trivial

## Mentor Conduct
- Provide constructive, actionable feedback.
- Avoid low-effort approvals (e.g., only "lgtm").
- Use priority labels responsibly.

## Review Quality Scoring
- Detailed reviews receive full weight.
- Low-effort reviews receive reduced weight.

## Escalation
Escalate suspicious PRs via `needs-human-review`, never via hostile comments.
