## 📝 Description
This PR addresses accessibility and keyboard trap issues for modals across both `index.html` and `app.js` code paths. It implements the focus restoration mechanism strictly using `document.activeElement` for syncing both paths natively. The existing `trapFocus()` logic has been extended to gracefully handle the `Escape` key natively across all modals, negating the need for any overly complex external logic.
Additionally, it adds the missing `role="dialog"`, `aria-modal="true"`, and `aria-labelledby` attributes to `#orgModal` for screen readers. Unrelated changes (like CSS formatting) have been strictly separated and removed.
